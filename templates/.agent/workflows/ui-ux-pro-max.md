# /ui-ux-pro-max - UI Design Workflow

## Overview

Use this workflow to turn a product request into a design direction before implementation. It pairs the ui-ux-pro-max skill with frontend implementation guidance.

## When to Use

- Designing a landing page or marketing site
- Creating a dashboard or admin UI
- Choosing visual direction for a feature or component set
- Auditing UI quality before implementation

## Workflow

### Step 1: Extract Design Inputs

- Product type
- Industry
- Audience
- Visual tone
- Stack or implementation target

### Step 2: Choose the Design System Direction

- Style family
- Color palette
- Typography pairing
- Layout pattern
- Motion and accessibility constraints

### Step 3: Map Design to Implementation

- Which components are needed?
- What responsive states matter?
- What empty, loading, error, and success states are required?

### Step 4: Review Quality

- Contrast and readability
- Hierarchy and spacing
- Touch and keyboard accessibility
- Visual consistency across screens

## Output

- [ ] Design direction defined
- [ ] Palette and typography chosen
- [ ] Layout pattern selected
- [ ] Implementation constraints noted
